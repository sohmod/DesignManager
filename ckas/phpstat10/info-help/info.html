Information About PHPstat
PHPstat is a web statistic program that does not require the use of a database.
The program registers number of visitors, unique visitors (ip pr. day), color depth, screen size, browser, javascript support, operating systems, country (top domain) and referrers.
The script is easy to install and the setup is done by web, and includes layout, css and password protection.
! Remember that as this script does not use a database, a large website with many visitors will increase the server's workload.
Recommend a maximum of 1000 unique visitors a day depending on the server speed.
PHPstat works with php 4.1 Older versions have not been tested. See F.A.Q for more info.

Installation of PHPstat
Unzip PHPstat and save it in a directory on the server (e.g. "/phpstat").
! Remember to change the file rights in the directory "data/*.*" (666).
! In the file you wish to have statistics on (e.g. index.html) put the following code in your file. 

<!-- PHPstat code -->
<script language="JavaScript">
var ref=""+escape(top.document.referrer); var colord = window.screen.colorDepth; 
var res = window.screen.width + "x" + window.screen.height;
document.write("<img src='http://www.yoursite.com/phpstat/stat.php?referer=" + ref + "&js=yes&color=" + colord + "&res=" + res + "' border=0>");
</script><noscript><img src="http://www.yoursite.com/phpstat/stat.php" border="0"></noscript>
<!-- PHPstat code -->
! Remember to change the 2 <img> tags to match your website (e.g. http://www.yoursite.com/phpstat/stat.php):
Since an image tag "<img>" calls the script, there will be a 1x1pixel transparent gif inserted on the page.
Remember to reset your stats on use to get the correct start date.
To see your stats, use the following url: "http://www.yoursite.com/phpstat".

Setup of PHPstat
Setup is done from the web - Go to "Settings" and press "PHPstat Setup".
! Username and Password are initially set to "user" / "abc123"

Frequently Asked Questions
Q:
I Get this error: "...Warning: fopen("data/filename", "w")..."?
A:
Change the file permission to "chmode 666" in "/data/*.*"

Q:
Chmod doesn't work?
A:
Some windows ftp clients don't support chmode. Use a program like WS_Ftp

Q:
My script doesn't register statistics?
A:
Remember to put the JavaScript code on your page and change the path to match your website

Q:
My script shows all hits as "0"?
A:
A few have reported this error when running PHPstat on a WinNT/IIS server. The problem is caused by the function "flock()". A way to solve this is removing the function. This may cause problems if 2 people are visiting your website at the exact same time

Q:
My webserver is using a php version below 4.1 - What do I do?
A:
Upgrade... If this is not possible, open all .php files and change "$_REQUEST['var']" with "$var"

Q:
I still haven't solved my problem?
A:
Check for updates on the webpage below. If you still have problems contact me. If possible please include your webserver's operating system and php version

Q:
I don't know my webserver's operating system or php version?
A:
Use the php function "phpinfo()". It will tell you all you need to know

Q:

Yes. You can do practically anything you want. Remember to read the license agreement
